## Design instructions
- The framework to use is slidev
- Preferred slide layouts: `default`, `two-cols`, `image-right`
- Keep a lean, clean, minimalistic design
- Slidev supports "Rough Notation" for emphasis, which should be used when highlighting.
- The color scheme schould be minimal, totalling at most two colors (plus shades) throughout the design.
- Allowed font-familiies: Roboto, Roboto Mono, Roboto Condensed.
- Check for overflows and misaligned content.

## Dev environment tips
- Do not use any `npm` commands to be run in the terminal.
- Slide content lives in the file `slides.md`.
 
## Testing instructions
- The presentation is hosted at http://localhost:3030, use a tool to acess it.
- There is no need to start a new instance of the presentation at any time.
- Never takes screenshots of any type.
