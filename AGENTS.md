## Dev environment tips
- The presentation is hosted at http://localhost:3030, use the MCP to access it and check the output.
- Do not use any `npm` commands to be run in the terminal.
- Slide content lives in the file `slides.md`.
- There is no need to start a new instance of the presentation at any time.
- Never takes screenshots of any type.

## Design instructions
- The framework to use is slidev
- Preferred slide layouts: `default`, `two-cols-title`, `image-right`
- Keep a lean, clean, minimalistic design
- The color scheme schould be minimal, totalling at most two colors (plus shades) throughout the design. Prefer to use the color scheme given by the theme.
- Preferred font-familiies: Roboto, Roboto Mono, Roboto Condensed.
- Check for overflows and misaligned content.
- If possible use the neversink theme. An example presentation can be found here: https://gureckis.github.io/slidev-theme-neversink/example/
